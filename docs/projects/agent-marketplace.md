## Overview

A **curated, monetized marketplace** for agents, MCP servers, and workflows — with
trust, security review, and revenue share. Listings are security-reviewed and
install in one click into real workflows via ContextOS, with usage-based revenue
share for authors.

## Problem

Plugin stores (GPT Store) have weak quality control, monetization, and discovery;
MCP registries are listings, not curated/monetized marketplaces; Hugging Face is
model/space-focused, not agent/workflow commerce.

## Differentiators

1. **Integrated with ContextOS** — one-click install into real workflows.
2. **Security-reviewed listings.**
3. **Usage-based revenue share.**

## Competitive landscape

GPT Store / plugin stores (distribution, weak curation/monetization), MCP
registries (discovery, not monetized marketplace), Hugging Face (models/spaces,
not workflow commerce).

## Moat

Distribution + trust. Only valuable once a platform sends it traffic — so it
compounds with ContextOS adoption.

## Risks

**Cold-start is the whole game.** Mitigation: **build last (Year 3)**, after
ContextOS, the MCP Server Generator, and Agent Monitoring create supply and
distribution.

## Role in the platform family

The **monetization flywheel** — built last, riding the distribution and trust the
rest of the family establishes.
