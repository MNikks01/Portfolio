## Overview

An education platform connecting schools, teachers, parents, and students —
real-time announcements, academic updates, fees, and media sharing. Mobile-first,
built and run solo from zero to launch.

## Problem

Schools, parents, and students were stuck with fragmented, offline communication
— paper circulars, scattered WhatsApp groups, no single source of truth for
attendance, announcements, fees, and academics. Coordination was slow and
information got lost.

## Solution

A mobile-first platform (app + web dashboard) that centralizes school–parent–
student communication around the workflows administrators actually use:
real-time announcements, academic updates, and media sharing.

## Tech Stack

React · React Native · Node.js · Express · GraphQL · Firebase · Cloudinary ·
Socket.IO

## Architecture

- React Native mobile app + React web dashboard (shared API).
- Node.js + Express + GraphQL backend.
- Firebase for auth and push notifications.
- Cloudinary for media storage/transformation.
- Socket.IO for real-time messaging/announcements.

## Challenges

- Building and maintaining web + mobile + backend as a single person.
- Earning trust with non-technical school buyers.
- Translating technical capability into business value during demos.

## Achievements

- Validated the idea with administrators **before** building.
- Shipped to the **Google Play Store**.
- Presented/demoed to **100+ school administrators**.
- Visited **50+ schools** for in-person customer discovery.
- Owned the full loop: idea → validation → design → engineering → launch → sales
  → iteration.

## Lessons Learned

Building is the easy part — distribution, trust, and understanding the buyer make
a product real. Validate before building, sell what you build, and stay close to
the customer.
