# Prompt — Architecture review

```
Review the architecture for scalability and maintainability.

Assess:
- Module boundaries & separation of concerns.
- Duplication (e.g. repeated card markup → shared SectionCard).
- Folder structure consistency.
- Design-token system integrity.
- Resilience (error boundaries, graceful degradation).
- Dependency hygiene & bundle impact.

Output: findings + a prioritized, incremental refactor plan.
Record any significant decision as an ADR in .ai/decisions/ and
docs/architecture/architecture-decisions.md. Add debt/backlog items as needed.
```
