# ADR-003 — CSS-variable design tokens for theming

- **Status:** Accepted
- **Date:** 2026-06-17

## Context

The site is dark-first (neon, glassmorphism, translucent-white overlays). Adding
a light theme by editing every component would be error-prone and unmaintainable.

## Decision

Drive all colors through **CSS variables** defined once in `globals.css`
(`:root` = dark default, `.light` = light override), stored as RGB channels so
Tailwind alpha modifiers work. Map them to semantic Tailwind colors
(`ink/fg/soft/muted/faint`, `overlay`, `bg-deep/panel`). Toggle via a `light`
class on `<html>` (persisted, `prefers-color-scheme`, applied pre-paint).
Light-only tweaks use a custom `light:` Tailwind variant.

## Consequences

- Both themes work without per-component overrides.
- **Hard rule:** never use literal `white`/`zinc` classes — they break light mode.
- Brand accents stay constant across themes.
