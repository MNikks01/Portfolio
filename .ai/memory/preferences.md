# Memory — Preferences

How Nikhil likes work done (observed across sessions).

## Engineering

- High polish: "Apple-level polish, Stripe-level design, Linear-level animations."
- Production-grade by default — tests, CI, governance, docs.
- Verify changes (build + screenshots) before claiming done; be honest about
  what's verified vs. assumed.
- Don't redesign existing visual identity unless asked.

## Communication

- Wants concise summaries of what changed, plus honest caveats (estimates,
  unverified bits, residual risks).
- Appreciates flagging risky/large refactors rather than doing them silently.

## Workflow

- Commit + push to `https://github.com/MNikks01/Portfolio` (branch `main`); the
  app at repo root; Netlify auto-deploys.
- Keep `/docs` canonical; update docs first, then code.
- Display name on the public site is "Nikhil" (no surname).

## Conventions

- Semantic color tokens only; both themes must work.
- Co-locate tests; keep quality gate green.
